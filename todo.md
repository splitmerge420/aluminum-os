# Cross-Platform UX Audit — Aluminum OS

## Phase 1: Visual & Interaction Audit
- [ ] Audit all components from each platform user's perspective
- [ ] Document all friction points

## Phase 2: Global UX Fixes
- [ ] Touch targets: minimum 44px for iOS/Android (Apple HIG + Material)
- [ ] Scrolling: momentum scroll, overscroll behavior, scroll containers
- [ ] Keyboard navigation: focus rings, tab order, escape to close
- [ ] Responsive breakpoints: mobile (320-480), tablet (768), laptop (1024), desktop (1440+)
- [ ] Font sizing: minimum 14px body, 12px minimum for any text
- [ ] Contrast ratios: WCAG AA minimum (4.5:1 body, 3:1 large)
- [ ] Click/tap feedback: hover states, active states, touch ripples
- [ ] Safe areas: notch, home indicator, status bar padding

## Phase 3: Platform-Specific Patterns
- [ ] macOS: window traffic lights, trackpad gestures, menu bar feel
- [ ] Windows: snap layouts feel, taskbar familiarity, right-click menus
- [ ] ChromeOS: launcher feel, shelf behavior, PWA-like experience
- [ ] Android: Material You patterns, back gesture, bottom nav familiarity
- [ ] iOS: swipe gestures, safe areas, haptic feedback patterns, bottom sheet
- [ ] Pixel OS: At-a-Glance widget feel, Material You dynamic color

## Phase 4: Shell Chrome Polish
- [ ] Dock: overflow handling, tooltip positioning, touch-friendly sizing
- [ ] TopBar: status icons, clock, responsive collapse
- [ ] Window chrome: drag handles, resize, close/min/max buttons, mobile fullscreen
- [ ] Context menu: touch-hold trigger, positioning near edges
- [ ] App Launcher: search UX, category filtering, keyboard shortcuts

## Phase 5: App-Level Fixes
- [ ] All 23 apps: scroll containers, text truncation, responsive layouts
- [ ] WishListApp: grid responsiveness, filter UX, detail panel
- [ ] CouncilApp: session dialogue readability, member cards
- [ ] VaultApp: card grid, search, category pills
- [ ] GovernanceApp: table responsiveness, audit log scroll
